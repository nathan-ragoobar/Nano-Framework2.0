\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename \+\_\+\+Matrix\+Type, typename \+\_\+\+Preconditioner$>$\newline
class Eigen\+::\+IDRS$<$ \+\_\+\+Matrix\+Type, \+\_\+\+Preconditioner $>$}
The Induced Dimension Reduction method (IDR(s)) is a short-\/recurrences Krylov method for sparse square problems. 

This class allows to solve for A.\+x = b sparse linear problems. The vectors x and b can be either dense or sparse. he Induced Dimension Reduction method, IDR(), is a robust and efficient short-\/recurrence Krylov subspace method for solving large nonsymmetric systems of linear equations.

For indefinite systems IDR(\+S) outperforms both Bi\+CGStab and Bi\+CGStab(\+L). Additionally, IDR(\+S) can handle matrices with complex eigenvalues more efficiently than Bi\+CGStab.

Many problems that do not converge for \doxylink{classEigen_1_1BiCGSTAB}{Bi\+CGSTAB} converge for IDR(s) (for larger values of s). And if both methods converge the convergence for IDR(s) is typically much faster for difficult systems (for example indefinite problems).

IDR(s) is a limited memory finite termination method. In exact arithmetic it converges in at most N+\+N/s iterations, with N the system size. It uses a fixed number of 4+3s vector. In comparison, \doxylink{classEigen_1_1BiCGSTAB}{Bi\+CGSTAB} terminates in 2N iterations and uses 7 vectors. \doxylink{classEigen_1_1GMRES}{GMRES} terminates in at most N iterations, and uses I+3 vectors, with I the number of iterations. Restarting \doxylink{classEigen_1_1GMRES}{GMRES} limits the memory consumption, but destroys the finite termination property.

The maximal number of iterations and tolerance value can be controlled via the \doxylink{classEigen_1_1IterativeSolverBase_a1bbaf86b7df6d54e78ced4da6ed2dac1}{set\+Max\+Iterations()} and \doxylink{classEigen_1_1IterativeSolverBase_ac43c50c69a5b621cb5a9991a1005750d}{set\+Tolerance()} methods. The defaults are the size of the problem for the maximal number of iterations and \doxylink{structEigen_1_1GenericNumTraits_a3a573c69713fbb143fb49af7db463cee}{Num\+Traits$<$\+Scalar$>$\+::epsilon()} for the tolerance.

The tolerance corresponds to the relative residual error\+: \texorpdfstring{$\vert$}{|}\+Ax-\/b\texorpdfstring{$\vert$}{|}/\texorpdfstring{$\vert$}{|}b\texorpdfstring{$\vert$}{|}

{\bfseries{Performance\+:}} when using sparse matrices, best performance is achied for a row-\/major sparse matrix format. Moreover, in this case multi-\/threading can be exploited if the user code is compiled with Open\+MP enabled. See Topic\+Multi\+Threading for details.

By default the iterations start with x=0 as an initial guess of the solution. One can control the start using the \doxylink{classEigen_1_1IterativeSolverBase_ad8b889910ed260591652f22c640d6a20}{solve\+With\+Guess()} method.
