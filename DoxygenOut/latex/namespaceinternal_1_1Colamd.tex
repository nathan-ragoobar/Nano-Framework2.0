set default parameters The use of this routine is optional. 

\doxylink{namespaceinternal_1_1Colamd}{Colamd}\+: rows with more than (knobs \mbox{[}Dense\+Row\mbox{]} \texorpdfstring{$\ast$}{*} n\+\_\+col) entries are removed prior to ordering. Columns with more than (knobs \mbox{[}Dense\+Col\mbox{]} \texorpdfstring{$\ast$}{*} n\+\_\+row) entries are removed prior to ordering, and placed last in the output column ordering.

Dense\+Row and Dense\+Col are defined as 0 and 1, respectively, in colamd.\+h. Default values of these two knobs are both 0.\+5. Currently, only knobs \mbox{[}0\mbox{]} and knobs \mbox{[}1\mbox{]} are used, but future versions may use more knobs. set default parameters The use of this routine is optional. 

\doxylink{namespaceinternal_1_1Colamd}{Colamd}\+: rows with more than (knobs \mbox{[}Dense\+Row\mbox{]} \texorpdfstring{$\ast$}{*} n\+\_\+col) entries are removed prior to ordering. Columns with more than (knobs \mbox{[}Dense\+Col\mbox{]} \texorpdfstring{$\ast$}{*} n\+\_\+row) entries are removed prior to ordering, and placed last in the output column ordering.

Dense\+Row and Dense\+Col are defined as 0 and 1, respectively, in colamd.\+h. Default values of these two knobs are both 0.\+5. Currently, only knobs \mbox{[}0\mbox{]} and knobs \mbox{[}1\mbox{]} are used, but future versions may use more knobs. If so, they will be properly set to their defaults by the future version of colamd\+\_\+set\+\_\+defaults, so that the code that calls colamd will not need to change, assuming that you either use colamd\+\_\+set\+\_\+defaults, or pass a (double \texorpdfstring{$\ast$}{*}) NULL pointer as the knobs array to colamd or symamd.
