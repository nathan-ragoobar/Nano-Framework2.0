\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Xpr\+Type, typename Row\+Indices, typename Col\+Indices$>$\newline
class Eigen\+::\+Indexed\+View$<$ Xpr\+Type, Row\+Indices, Col\+Indices $>$}
Expression of a non-\/sequential sub-\/matrix defined by arbitrary sequences of row and column indices. 


\begin{DoxyTemplParams}{Template Parameters}
{\em Xpr\+Type} & the type of the expression in which we are taking the intersections of sub-\/rows and sub-\/columns \\
\hline
{\em Row\+Indices} & the type of the object defining the sequence of row indices \\
\hline
{\em Col\+Indices} & the type of the object defining the sequence of column indices\\
\hline
\end{DoxyTemplParams}
This class represents an expression of a sub-\/matrix (or sub-\/vector) defined as the intersection of sub-\/sets of rows and columns, that are themself defined by generic sequences of row indices $ \{r_0,r_1,..r_{m-1}\} $ and column indices $ \{c_0,c_1,..c_{n-1} \}$. Let $ A $ be the nested matrix, then the resulting matrix $ B $ has {\ttfamily m} rows and {\ttfamily n} columns, and its entries are given by\+: $ B(i,j) = A(r_i,c_j) $.

The {\ttfamily Row\+Indices} and {\ttfamily Col\+Indices} types must be compatible with the following API\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{<integral\ \mbox{\hyperlink{namespacefineweb_a8695301745cad3a63cfe73f8f67ebf3d}{type}}>\ operator[](\mbox{\hyperlink{namespaceEigen_a62e77e0933482dafde8fe197d9a2cfde}{Index}})\ \textcolor{keyword}{const};}
\DoxyCodeLine{\mbox{\hyperlink{namespaceEigen_a62e77e0933482dafde8fe197d9a2cfde}{Index}}\ \mbox{\hyperlink{abseil-cpp_2absl_2base_2internal_2low__level__alloc_8cc_aad9b71a31372d5c0ab9c23163efe9544}{size}}()\ \textcolor{keyword}{const};}

\end{DoxyCode}


Typical supported types thus include\+:
\begin{DoxyItemize}
\item std\+::vector$<$int$>$
\item std\+::valarray$<$int$>$
\item std\+::array$<$int$>$
\item Plain C arrays\+: int\mbox{[}N\mbox{]}
\item Eigen\+::\+Array\+Xi
\item decltype(\+Array\+Xi\+::\+Lin\+Spaced(...))
\item Any view/expressions of the previous types
\item \doxylink{classEigen_1_1ArithmeticSequence}{Eigen\+::\+Arithmetic\+Sequence}
\item \doxylink{structEigen_1_1internal_1_1AllRange}{Eigen\+::internal\+::\+All\+Range} (helper for \doxylink{namespaceEigen_ab8d46fedbcdd5cc49ca89b50ad2ba0e8}{Eigen\+::all})
\item \doxylink{structEigen_1_1internal_1_1SingleRange}{Eigen\+::internal\+::\+Single\+Range} (helper for single index)
\item etc.
\end{DoxyItemize}

In typical usages of Eigen, this class should never be used directly. It is the return type of \doxylink{classEigen_1_1DenseBase_a4ab85c7f3859f871bda589ee9cba93f2}{Dense\+Base\+::operator()(const Row\+Indices\&, const Col\+Indices\&)}.
