The number of rows {\itshape block\+Rows} and columns {\itshape block\+Cols} can also be specified at compile-\/time by passing \doxylink{namespaceEigen_a8636241003a5d035f1832765ba8e5418}{Eigen\+::fix$<$\+N$>$}, or Eigen\+::fix$<$\+N$>$(n) as arguments. Here is an example with a fixed number of rows {\ttfamily NRows} and dynamic number of columns {\ttfamily cols\+:}

This function thus fully covers the features offered by the following overloads \doxylink{BlockMethods_8h_a3ea80fa192d8755e555954f61ce5125e}{block$<$\+NRows,\+NCols$>$(\+Index, Index)}, and \doxylink{BlockMethods_8h_af200d4ec5239d24e2de39ae742762fdb}{block$<$\+NRows,\+NCols$>$(\+Index, Index, Index, Index)} that are thus obsolete. but with less redundancy and more consistency as it does not modify the argument order and seamlessly enable hybrid fixed/dynamic sizes.

\begin{DoxyNote}{Note}
Even in the case that the returned expression has dynamic size, in the case when it is applied to a fixed-\/size matrix, it inherits a fixed maximal size, which means that evaluating it does not cause a dynamic memory allocation.

The compile-\/time and run-\/time information should not contradict.
