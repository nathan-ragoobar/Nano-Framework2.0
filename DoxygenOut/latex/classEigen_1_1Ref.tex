\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Plain\+Object\+Type, int Options = 0, typename Stride\+Type = typename internal\+::conditional$<$\+Plain\+Object\+Type\+::\+Is\+Vector\+At\+Compile\+Time,\+Inner\+Stride$<$1$>$,\+Outer\+Stride$<$$>$ $>$\+::type$>$\newline
class Eigen\+::\+Ref$<$ Plain\+Object\+Type, Options, Stride\+Type $>$}
A matrix or vector expression mapping an existing expression. 


\begin{DoxyTemplParams}{Template Parameters}
{\em Plain\+Object\+Type} & the equivalent matrix type of the mapped data \\
\hline
{\em Options} & specifies the pointer alignment in bytes. It can be\+: {\ttfamily \doxylink{group__enums_gga45fe06e29902b7a2773de05ba27b47a1a86400b6f9ebc01d9997b12512f3ea5fc}{Aligned128}}, , {\ttfamily \doxylink{group__enums_gga45fe06e29902b7a2773de05ba27b47a1a0f4f4451618a2e67cade6b54ca0fc84b}{Aligned64}}, {\ttfamily \doxylink{group__enums_gga45fe06e29902b7a2773de05ba27b47a1a7797d247b86f6e9837cc338beb6060de}{Aligned32}}, {\ttfamily \doxylink{group__enums_gga45fe06e29902b7a2773de05ba27b47a1af8e2bf74b04c02199f62c5e3c06dbfcc}{Aligned16}}, {\ttfamily \doxylink{group__enums_gga45fe06e29902b7a2773de05ba27b47a1ae9267ee4260c25c5b162ae72bf18a7bd}{Aligned8}} or {\ttfamily \doxylink{group__enums_gga45fe06e29902b7a2773de05ba27b47a1ac935220b4c844108e183ebe30a4d5204}{Unaligned}}. The default is {\ttfamily \doxylink{group__enums_gga45fe06e29902b7a2773de05ba27b47a1ac935220b4c844108e183ebe30a4d5204}{Unaligned}}. \\
\hline
{\em Stride\+Type} & optionally specifies strides. By default, \doxylink{classEigen_1_1Ref}{Ref} implies a contiguous storage along the inner dimension (inner stride==1), but accepts a variable outer stride (leading dimension). This can be overridden by specifying strides. The type passed here must be a specialization of the \doxylink{classEigen_1_1Stride}{Stride} template, see examples below.\\
\hline
\end{DoxyTemplParams}
This class provides a way to write non-\/template functions taking \doxylink{namespaceEigen}{Eigen} objects as parameters while limiting the number of copies. A Ref$<$$>$ object can represent either a const expression or a l-\/value\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{comment}{//\ in-\/out\ argument:}}
\DoxyCodeLine{\textcolor{keywordtype}{void}\ foo1(Ref<VectorXf>\ x);}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{comment}{//\ read-\/only\ const\ argument:}}
\DoxyCodeLine{\textcolor{keywordtype}{void}\ foo2(\textcolor{keyword}{const}\ Ref<const\ VectorXf>\&\ x);}

\end{DoxyCode}


In the in-\/out case, the input argument must satisfy the constraints of the actual Ref$<$$>$ type, otherwise a compilation issue will be triggered. By default, a Ref$<$\+Vector\+Xf$>$ can reference any dense vector expression of float having a contiguous memory layout. Likewise, a Ref$<$\+Matrix\+Xf$>$ can reference any column-\/major dense matrix expression of float whose column\textquotesingle{}s elements are contiguously stored with the possibility to have a constant space in-\/between each column, i.\+e. the inner stride must be equal to 1, but the outer stride (or leading dimension) can be greater than the number of rows.

In the const case, if the input expression does not match the above requirement, then it is evaluated into a temporary before being passed to the function. Here are some examples\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{MatrixXf\ \mbox{\hyperlink{abseil-cpp_2absl_2base_2internal_2fast__type__id__test_8cc_a1ea955b94ea8dc35e689055712257459}{A}};}
\DoxyCodeLine{VectorXf\ \mbox{\hyperlink{abseil-cpp_2absl_2container_2internal_2common__policy__traits__test_8cc_aa4c2a5552e9bc49b1816ff532f558c74}{a}};}
\DoxyCodeLine{foo1(\mbox{\hyperlink{abseil-cpp_2absl_2container_2internal_2common__policy__traits__test_8cc_aa4c2a5552e9bc49b1816ff532f558c74}{a}}.head());\ \ \ \ \ \ \ \ \ \ \ \ \ \textcolor{comment}{//\ OK}}
\DoxyCodeLine{foo1(A.col());\ \ \ \ \ \ \ \ \ \ \ \ \ \ \textcolor{comment}{//\ OK}}
\DoxyCodeLine{foo1(A.row());\ \ \ \ \ \ \ \ \ \ \ \ \ \ \textcolor{comment}{//\ Compilation\ error\ because\ here\ innerstride!=1}}
\DoxyCodeLine{foo2(A.row());\ \ \ \ \ \ \ \ \ \ \ \ \ \ \textcolor{comment}{//\ Compilation\ error\ because\ A.row()\ is\ a\ 1xN\ object\ while\ foo2\ is\ expecting\ a\ Nx1\ object}}
\DoxyCodeLine{foo2(A.row().transpose());\ \ \textcolor{comment}{//\ The\ row\ is\ copied\ into\ a\ contiguous\ temporary}}
\DoxyCodeLine{foo2(2*\mbox{\hyperlink{abseil-cpp_2absl_2container_2internal_2common__policy__traits__test_8cc_aa4c2a5552e9bc49b1816ff532f558c74}{a}});\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \textcolor{comment}{//\ The\ expression\ is\ evaluated\ into\ a\ temporary}}
\DoxyCodeLine{foo2(A.col().segment(2,4));\ \textcolor{comment}{//\ No\ temporary}}

\end{DoxyCode}


The range of inputs that can be referenced without temporary can be enlarged using the last two template parameters. Here is an example accepting an innerstride!=1\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{comment}{//\ in-\/out\ argument:}}
\DoxyCodeLine{\textcolor{keywordtype}{void}\ foo3(Ref<VectorXf,0,InnerStride<>\ >\ x);}
\DoxyCodeLine{foo3(A.row());\ \ \ \ \ \ \ \ \ \ \ \ \ \ \textcolor{comment}{//\ OK}}

\end{DoxyCode}
 The downside here is that the function foo3 might be significantly slower than foo1 because it won\textquotesingle{}t be able to exploit vectorization, and will involve more expensive address computations even if the input is contiguously stored in memory. To overcome this issue, one might propose to overload internally calling a template function, e.\+g.\+:
