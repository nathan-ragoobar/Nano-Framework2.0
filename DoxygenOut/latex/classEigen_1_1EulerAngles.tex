\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename \+\_\+\+Scalar, class \+\_\+\+System$>$\newline
class Eigen\+::\+Euler\+Angles$<$ \+\_\+\+Scalar, \+\_\+\+System $>$}
Represents a rotation in a 3 dimensional space as three Euler angles. 

Euler rotation is a set of three rotation of three angles over three fixed axes, defined by the \doxylink{classEigen_1_1EulerSystem}{Euler\+System} given as a template parameter.

Here is how intrinsic Euler angles works\+:
\begin{DoxyItemize}
\item first, rotate the axes system over the alpha axis in angle alpha
\item then, rotate the axes system over the beta axis(which was rotated in the first stage) in angle beta
\item then, rotate the axes system over the gamma axis(which was rotated in the two stages above) in angle gamma
\end{DoxyItemize}

\begin{DoxyNote}{Note}
This class support only intrinsic Euler angles for simplicity, see \doxylink{classEigen_1_1EulerSystem}{Euler\+System} how to easily overcome this for extrinsic systems.
\end{DoxyNote}
\hypertarget{classEigen_1_1EulerAngles_autotoc_md43}{}\doxysubsubsubsection{\texorpdfstring{Rotation representation and conversions}{Rotation representation and conversions}}\label{classEigen_1_1EulerAngles_autotoc_md43}
It has been proved(see Wikipedia link below) that every rotation can be represented by Euler angles, but there is no single representation (e.\+g. unlike rotation matrices). Therefore, you can convert from \doxylink{namespaceEigen}{Eigen} rotation and to them (including rotation matrices, which is not called "{}rotations"{} by \doxylink{namespaceEigen}{Eigen} design).

Euler angles usually used for\+:
\begin{DoxyItemize}
\item convenient human representation of rotation, especially in interactive GUI.
\item gimbal systems and robotics
\item efficient encoding(i.\+e. 3 floats only) of rotation for network protocols.
\end{DoxyItemize}

However, Euler angles are slow comparing to quaternion or matrices, because their unnatural math definition, although it\textquotesingle{}s simple for human. To overcome this, this class provide easy movement from the math friendly representation to the human friendly representation, and vise-\/versa.

All the user need to do is a safe simple C++ type conversion, and this class take care for the math. Additionally, some axes related computation is done in compile time.\hypertarget{classEigen_1_1EulerAngles_autotoc_md44}{}\doxysubsubsubsubsection{\texorpdfstring{Euler angles ranges in conversions}{Euler angles ranges in conversions}}\label{classEigen_1_1EulerAngles_autotoc_md44}
Rotations representation as \doxylink{classEigen_1_1EulerAngles}{Euler\+Angles} are not single (unlike matrices), and even have infinite \doxylink{classEigen_1_1EulerAngles}{Euler\+Angles} representations.~\newline
 For example, add or subtract 2\texorpdfstring{$\ast$}{*}\+PI from either angle of \doxylink{classEigen_1_1EulerAngles}{Euler\+Angles} and you\textquotesingle{}ll get the same rotation. This is the general reason for infinite representation, but it\textquotesingle{}s not the only general reason for not having a single representation.

When converting rotation to \doxylink{classEigen_1_1EulerAngles}{Euler\+Angles}, this class convert it to specific ranges When converting some rotation to \doxylink{classEigen_1_1EulerAngles}{Euler\+Angles}, the rules for ranges are as follow\+:
\begin{DoxyItemize}
\item If the rotation we converting from is an \doxylink{classEigen_1_1EulerAngles}{Euler\+Angles} (even when it represented as \doxylink{classEigen_1_1RotationBase}{Rotation\+Base} explicitly), angles ranges are {\bfseries{undefined}}.
\item otherwise, alpha and gamma angles will be in the range \mbox{[}-\/PI, PI\mbox{]}.~\newline
 As for Beta angle\+:
\begin{DoxyItemize}
\item If the system is Tait-\/\+Bryan, the beta angle will be in the range \mbox{[}-\/PI/2, PI/2\mbox{]}.
\item otherwise\+:
\begin{DoxyItemize}
\item If the beta axis is positive, the beta angle will be in the range \mbox{[}0, PI\mbox{]}
\item If the beta axis is negative, the beta angle will be in the range \mbox{[}-\/PI, 0\mbox{]}
\end{DoxyItemize}
\end{DoxyItemize}
\end{DoxyItemize}

Constructs and initialize Euler angles from a rotation {\ttfamily rot}.

\begin{DoxyNote}{Note}
If {\ttfamily rot} is an \doxylink{classEigen_1_1EulerAngles}{Euler\+Angles} (even when it represented as \doxylink{classEigen_1_1RotationBase}{Rotation\+Base} explicitly), angles ranges are {\bfseries{undefined}}. Otherwise, alpha and gamma angles will be in the range \mbox{[}-\/PI, PI\mbox{]}.~\newline
 As for Beta angle\+:
\begin{DoxyItemize}
\item If the system is Tait-\/\+Bryan, the beta angle will be in the range \mbox{[}-\/PI/2, PI/2\mbox{]}.
\item otherwise\+:
\begin{DoxyItemize}
\item If the beta axis is positive, the beta angle will be in the range \mbox{[}0, PI\mbox{]}
\item If the beta axis is negative, the beta angle will be in the range \mbox{[}-\/PI, 0\mbox{]} 
\end{DoxyItemize}
\end{DoxyItemize}
\end{DoxyNote}
