\doxysubsection{Detailed Description}
\subsubsection*{template$<$int \+\_\+\+Alpha\+Axis, int \+\_\+\+Beta\+Axis, int \+\_\+\+Gamma\+Axis$>$\newline
class Eigen\+::\+Euler\+System$<$ \+\_\+\+Alpha\+Axis, \+\_\+\+Beta\+Axis, \+\_\+\+Gamma\+Axis $>$}
Represents a fixed Euler rotation system. 

This meta-\/class goal is to represent the Euler system in compilation time, for \doxylink{classEigen_1_1EulerAngles}{Euler\+Angles}.

You can use this class to get two things\+:
\begin{DoxyItemize}
\item Build an Euler system, and then pass it as a template parameter to \doxylink{classEigen_1_1EulerAngles}{Euler\+Angles}.
\item Query some compile time data about an Euler system. (e.\+g. Whether it\textquotesingle{}s Tait-\/\+Bryan)
\end{DoxyItemize}

Euler rotation is a set of three rotation on fixed axes. (see \doxylink{classEigen_1_1EulerAngles}{Euler\+Angles}) This meta-\/class store constantly those signed axes. (see \doxylink{namespaceEigen_ae614aa7cdd687fb5c421a54f2ce5c361}{Euler\+Axis})\hypertarget{classEigen_1_1EulerSystem_autotoc_md48}{}\doxysubsubsubsection{\texorpdfstring{Types of Euler systems}{Types of Euler systems}}\label{classEigen_1_1EulerSystem_autotoc_md48}
All and only valid 3 dimension Euler rotation over standard signed axes\{+X,+Y,+Z,-\/X,-\/Y,-\/Z\} are supported\+:
\begin{DoxyItemize}
\item all axes X, Y, Z in each valid order (see below what order is valid)
\item rotation over the axis is supported both over the positive and negative directions.
\item both Tait-\/\+Bryan and proper/classic Euler angles (i.\+e. the opposite).
\end{DoxyItemize}

Since \doxylink{classEigen_1_1EulerSystem}{Euler\+System} support both positive and negative directions, you may call this rotation distinction in other names\+:
\begin{DoxyItemize}
\item {\itshape right handed} or {\itshape left handed}
\item {\itshape counterclockwise} or {\itshape clockwise}
\end{DoxyItemize}

Notice all axed combination are valid, and would trigger a static assertion. Same unsigned axes can\textquotesingle{}t be neighbors, e.\+g. \{X,X,Y\} is invalid. This yield two and only two classes\+:
\begin{DoxyItemize}
\item {\itshape Tait-\/\+Bryan} -\/ all unsigned axes are distinct, e.\+g. \{X,Y,Z\}
\item {\itshape proper/classic Euler angles} -\/ The first and the third unsigned axes is equal, and the second is different, e.\+g. \{X,Y,X\}
\end{DoxyItemize}\hypertarget{classEigen_1_1EulerSystem_autotoc_md49}{}\doxysubsubsubsection{\texorpdfstring{Intrinsic vs extrinsic Euler systems}{Intrinsic vs extrinsic Euler systems}}\label{classEigen_1_1EulerSystem_autotoc_md49}
Only intrinsic Euler systems are supported for simplicity. If you want to use extrinsic Euler systems, just use the equal intrinsic opposite order for axes and angles. I.\+e axes (A,B,C) becomes (C,B,A), and angles (a,b,c) becomes (c,b,a).\hypertarget{classEigen_1_1EulerSystem_autotoc_md50}{}\doxysubsubsubsection{\texorpdfstring{Convenient user typedefs}{Convenient user typedefs}}\label{classEigen_1_1EulerSystem_autotoc_md50}
Convenient typedefs for \doxylink{classEigen_1_1EulerSystem}{Euler\+System} exist (only for positive axes Euler systems), in a form of \doxylink{classEigen_1_1EulerSystem}{Euler\+System}\{A\}\{B\}\{C\}, e.\+g. Euler\+System\+XYZ.\hypertarget{classEigen_1_1EulerSystem_autotoc_md51}{}\doxysubsubsubsection{\texorpdfstring{Additional reading}{Additional reading}}\label{classEigen_1_1EulerSystem_autotoc_md51}
More information about Euler angles\+: \href{https://en.wikipedia.org/wiki/Euler_angles}{\texttt{ https\+://en.\+wikipedia.\+org/wiki/\+Euler\+\_\+angles}}
